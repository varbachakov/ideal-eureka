# Рабочая Концепция Ярми

## Рабочее Название

Платформа подтвержденной реальности.

Ярми - не соцсеть видео и не "TikTok без фильтров". Это потребительский продукт, который начинается как лента реальных моментов, но под капотом строит инфраструктуру доверия к происхождению цифрового контента.

Короткая формула:

```text
Реальное событие -> подтвержденная фиксация -> публикация -> история доверия
```

## Проблема

Интернет быстро движется к миру, где:

- фото генерируются ИИ;
- видео генерируются ИИ;
- отзывы пишут боты;
- личности подделываются;
- происхождение контента неизвестно.

Главный вопрос постепенно меняется:

```text
Не "что я вижу?", а "могу ли я доверять происхождению этого?"
```

## Решение

Ярми позволяет публиковать контент, снятый в моменте через приложение.

Основной принцип:

```text
Реальное событие -> подтвержденная фиксация -> публикация
```

Ярми подтверждает происхождение видео: оно снято внутри приложения, в непрерывной camera-сессии, без импорта, склейки, фильтров и AI-обработки в capture-flow.

Важно: Ярми не обещает абсолютную "истину" события. Пользователь может снять постановочную сцену или экран другого устройства. Платформа подтверждает provenance/capture-flow, а не доказывает философскую истинность происходящего.

## Формулировки

Для пользователя:

```text
Настоящие люди. Настоящие моменты.
```

Более точная продуктовая формулировка:

```text
Ярми показывает видео, снятые в моменте внутри приложения, без загрузки из галереи, фильтров, склеек и AI-обработки.
```

Для инвестора:

```text
Мы строим крупнейшую сеть подтвержденной реальности и происхождения цифрового контента в эпоху генеративного ИИ.
```

## Пользовательский Опыт

Пользователь:

1. Открывает приложение.
2. Сразу попадает в камеру.
3. Снимает видео одним непрерывным дублем.
4. Может обрезать начало и конец.
5. Публикует ролик или удаляет его.

Нельзя:

- загружать из галереи;
- склеивать видео;
- применять фильтры;
- изменять содержимое ролика;
- использовать AI-обработку в capture-flow.

Можно:

- снять видео внутри приложения;
- обрезать начало/конец;
- удалить;
- опубликовать;
- позже сжать и перекодировать видео для доставки зрителю.

## Что Видят Пользователи

Лента реальных моментов.

Не лучших моментов.  
Не только вирусных роликов.  
Не витрина продакшена.  

А настоящая жизнь:

- поездка на работу;
- прогулка;
- путешествие;
- концерт;
- утро;
- дождь;
- семейный ужин;
- место или событие прямо сейчас.

Ярми берет у сторис скорость, живость и социальное присутствие. По реакции продукт может быть близок к Telegram Stories: быстро, понятно, без лишнего трения. По развлекательности Ярми может смотреть в сторону легкости TikTok, но не должен терять принцип подтвержденного происхождения.

## Главный Объект Системы

Главный объект - не видео. Главный объект - `Capture Event`.

Видео - это медиафайл.  
`Capture Event` - это доказательство того, как этот медиафайл появился.

Для события фиксируется:

- id события;
- id автора;
- время начала записи;
- время окончания записи;
- длительность;
- hash оригинального видео;
- hash опубликованной версии;
- версия приложения;
- платформа: Android/iOS;
- параметры камеры;
- результат проверки непрерывности записи;
- факт отсутствия импорта из галереи;
- факт отсутствия склейки;
- факт отсутствия AI/edit pipeline;
- геолокация, если пользователь разрешил;
- дополнительные сигналы, если пользователь разрешил;
- подпись события.

## Контент И Доказательство

Хранить отдельно:

```text
Content Asset
- original video
- transcoded video
- thumbnail
- playback versions

Event Proof
- hashes
- signature
- capture metadata
- verification state
- trust signals
```

Видео можно:

- сжимать;
- перекодировать;
- оптимизировать;
- удалять по правилам хранения.

Доказательство события должно оставаться неизменным.

## Главный Актив Платформы

Главный актив - не просмотры, не лайки и не сами видео.

Главный актив - граф подтвержденных событий.

```text
Пользователь
  -> Событие #1
  -> Событие #2
  -> Событие #3
```

Со временем формируется история достоверности:

- пользователя;
- места;
- отзыва;
- события;
- сообщества;
- публикации.

## Verification

На старте лучше не показывать числовой `Verification Score`, чтобы не создать ложное ощущение абсолютной истины.

Для массового интерфейса лучше начинать с понятных уровней:

```text
Снято в Ярми
Подтвержденная съемка
Подтвержденная съемка + дополнительные сигналы
Высокая достоверность происхождения
```

Внутренний score может учитывать:

- ролик снят встроенной камерой;
- нет признаков импорта;
- запись непрерывная;
- hash оригинала совпадает;
- опубликованная версия связана с оригиналом;
- приложение подписало событие;
- устройство не выглядит подозрительно;
- аккаунт имеет историю нормальных событий;
- геолокация согласуется со временем, если разрешена;
- дополнительные сигналы согласуются между собой.

## Threat Model

Ярми защищается от:

- импорта из галереи;
- монтажа до публикации;
- AI-генерации файла внутри capture-flow;
- подмены опубликованного видео после съемки;
- потери связи между оригиналом и публикацией.

Ярми не может полностью защититься от:

- съемки чужого экрана;
- постановочной сцены;
- spoof геолокации;
- root/jailbreak;
- модифицированного клиента;
- физической съемки поддельного объекта.

Поэтому обещание продукта должно быть точным:

```text
Мы подтверждаем происхождение видео и целостность capture-flow.
```

А не:

```text
Мы доказываем абсолютную правду.
```

## Privacy

Геолокация, устройство и дополнительные сигналы - чувствительные данные.

Нужно разделять:

- публичные proof-данные;
- приватные proof-данные автора;
- серверные trust-сигналы.

Геолокация не должна быть обязательной. Базовая ценность Ярми - подтвержденный capture-flow. Локация и дополнительные сигналы только усиливают доверие, если пользователь разрешил их использовать.

## Краткосрочная Стратегия: 0-3 Года

Основной продукт:

- камера;
- лента реальных видео;
- профили;
- подписки;
- реакции;
- комментарии;
- сообщества;
- дневник жизни.

Главная ценность:

```text
Смотри реальные моменты реальных людей.
```

Технический фундамент:

- стабильный capture-flow;
- запрет галереи;
- hash оригинала;
- простое состояние "Снято в Ярми";
- подготовка к `Capture Event`.

## Среднесрочная Стратегия: 3-5 Лет

Появляются:

- подтвержденные профили;
- уровни доверия;
- подтвержденные публикации;
- подтвержденные отзывы;
- подтвержденные посещения мест;
- публичные proof-страницы.

Пример:

```text
Этот отзыв оставил пользователь с историей подтвержденных событий.
```

## Долгосрочная Стратегия: 5-10 Лет

Ярми становится инфраструктурой доверия.

Можно подтверждать:

- посещение событий;
- опыт пользователя;
- отзывы;
- участие в мероприятиях;
- происхождение контента.

Появляется внешний API доверия:

```text
Кто это снял?
Когда это снято?
Через какой capture-flow?
Было ли изменено?
Можно ли проверить происхождение?
```

## Монетизация

На старте:

- подписка;
- премиум-функции;
- облачное хранение архива;
- расширенная история подтвержденных моментов.

Позже:

- verified reviews;
- бизнес-профили мест;
- корпоративные решения;
- API проверки подлинности контента;
- инфраструктура цифровой репутации.

## Архитектурный Принцип

С самого начала не смешивать:

```text
Видео как media asset
```

и

```text
Доказательство события как immutable proof
```

На уровне текущего мобильного MVP это означает:

- не размазывать прямые вызовы камеры по приложению;
- держать камеру за абстракцией;
- проектировать запись как будущий `Capture Event`;
- не добавлять gallery/import даже как скрытую debug-функцию;
- не добавлять фильтры и AI pipeline в capture-flow.

В будущем стоит двигаться к совместимости с открытыми provenance-подходами вроде C2PA/Content Credentials, но не зависеть от них на старте.

## Итоговая Формулировка

Ярми - платформа подтвержденной реальности.

Сегодня это мобильное приложение для коротких реальных видео, снятых только через камеру внутри приложения. Пользователь открывает камеру, фиксирует момент, может минимально обрезать начало и конец, а затем публикует ролик в ленту настоящей жизни.

Под капотом каждая публикация должна становиться `Capture Event`: событием с hash оригинала, подписью, метаданными съемки и уровнем доверия. Видео можно оптимизировать для просмотра, но доказательство события остается неизменным.

Долгосрочно Ярми строит граф подтвержденных событий - основу цифровой репутации и доверия к происхождению контента в эпоху генеративного ИИ.
