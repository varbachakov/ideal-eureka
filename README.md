# Mobile Yarmy

Mobile Yarmy - мобильное приложение Ярми и первый capture-клиент платформы подтвержденной реальности.

Ярми начинается как лента коротких реальных видеоисторий, но под капотом строит инфраструктуру доверия к происхождению цифрового контента. Это не "соцсеть видео" и не "TikTok без фильтров". Главный продуктовый объект - не ролик сам по себе, а подтвержденное событие съемки.

Короткая формула:

```text
Реальное событие -> подтвержденная фиксация -> публикация -> история доверия
```

Полная рабочая концепция зафиксирована в `PRODUCT_CONCEPT.md`.

## Главное Обещание

Ярми подтверждает происхождение видео: ролик был снят внутри приложения, в непрерывной camera-сессии, без импорта из галереи, склейки, фильтров и AI-обработки в capture-flow.

Важно: Ярми не обещает абсолютную "истину" события. Пользователь может снять постановочную сцену или экран другого устройства. Платформа подтверждает происхождение и целостность capture-flow, а не философскую истинность происходящего.

Пользовательская формулировка:

```text
Настоящие люди. Настоящие моменты.
```

## Продуктовая Идея

Ярми берет у сторис скорость, присутствие и легкость: человек быстро показывает, где он находится, что делает и что происходит вокруг него прямо сейчас.

Но Ярми делает сторис строже:

- видео можно снять только через камеру внутри приложения;
- загрузка из галереи запрещена;
- внешний монтаж, фильтры и AI-обработка запрещены в capture-flow;
- обрезка начала/конца разрешена, потому что доказательство строится вокруг факта съемки в приложении, а не вокруг сохранения каждой сырой секунды.

В Telegram Stories можно выбрать фото или видео из галереи. В Ярми нельзя. Это ключевое отличие продукта.

## Что Видит Пользователь

Лента Ярми - это лента реальных моментов, а не витрина лучших или самых вирусных роликов.

Примеры:

- дорога на работу;
- прогулка;
- путешествие;
- концерт;
- утро;
- дождь;
- семейный ужин;
- место или событие прямо сейчас.

По реакции Ярми стремится к простоте Telegram Stories: быстро, понятно, без лишнего трения. По развлекательности Ярми может смотреть в сторону легкости TikTok, но не должен превращаться в бесконечную ленту, которая уничтожает принцип подтвержденного происхождения.

## Capture Event

Каждая публикация в будущем должна становиться `Capture Event` - событием фиксации.

Для события могут фиксироваться:

- время записи;
- hash оригинального видео;
- hash опубликованной версии;
- длительность;
- версия приложения;
- платформа и параметры камеры;
- факт непрерывной записи;
- факт отсутствия импорта из галереи;
- факт отсутствия склейки и AI/edit pipeline;
- геолокация, если пользователь разрешил;
- дополнительные сигналы, если пользователь разрешил;
- криптографическая подпись события.

Архитектурно нужно разделять:

```text
Content Asset
- original video
- transcoded video
- thumbnail
- playback versions

Event Proof
- hashes
- signature
- capture metadata
- verification state
- trust signals
```

Видео можно сжимать, перекодировать и оптимизировать. Доказательство события должно оставаться неизменным.

## MVP

- Длина видео: до 60 секунд.
- Запись: один непрерывный дубль без паузы.
- Первый экран: камера.
- После записи: предпросмотр, обрезка начала/конца, действия "Опубликовать" и "Удалить".
- Если пользователь не авторизован, вход запрашивается только перед публикацией.
- Загрузка из галереи, фильтры, AI-обработка и внешний монтаж не входят в MVP.
- Просмотры, лайки и комментарии не входят в MVP, но это важное направление развития историй.

## UI Стек

- Forui - кнопки, иконки, диалоги, sheets, toasts и общая UI-система.
- Custom Yarmy UI - камера, record button, progress ring, camera overlay и trim.
- `flutter_animate` - пульсация, переходы, реакции и появление элементов.

## Текущее Состояние

Сейчас убран шаблонный counter screen, собран app shell и feature-структура `features/camera_capture`. Подключен Flutter `camera` plugin, добавлены разрешения камеры/микрофона для Android и iOS, первый экран открывает live camera preview и показывает русские состояния загрузки/ошибок.

Добавлена запись видео одним дублем до 60 секунд: экран показывает таймер, progress ring и автоматически останавливает запись на лимите. Для камеры добавлен fallback качества `high -> medium -> low`, чтобы эмуляторы и устройства с ограниченными camera surface combinations могли открыть preview.

Следующий этап - добавить экран предпросмотра снятого видео.
